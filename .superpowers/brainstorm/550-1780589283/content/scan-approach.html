¿Cómo escanea el usuario el equipo?

Define el flujo completo de captura. Elige el que más se ajusta a cómo son las etiquetas físicas en tu bodega.

A

Multi-código: escanear una vez, llenar varios campos

La cámara detecta todos los códigos de barras visibles en el frame al mismo tiempo. Cada valor escaneado se enruta al campo correcto por su patrón: 15 dígitos → IMEI, formato asset tag → Placa, etc. El usuario apunta la cámara a la etiqueta/caja y todos los campos disponibles se llenan automáticamente.

Pro

  • Un solo gesto = múltiples campos llenos
  • Funciona con etiquetas estándar existentes
  • No requiere preparar QR especiales

Contra

  • Solo llena campos que tienen código de barras (no texto libre)
  • Modelo/RAM/disco quedan manuales
B

OCR: foto de etiqueta → extracción de texto

El usuario apunta la cámara a cualquier etiqueta del equipo (sticker de serial, caja del celular). El sistema captura la imagen y usa Tesseract.js (OCR en browser) para leer el texto. Detecta patrones: "S/N:", "IMEI:", "RAM:", "GB", marca/modelo por palabras clave.

Pro

  • Lee texto impreso: modelo, procesador, RAM, disco
  • Funciona con cualquier etiqueta sin código de barras
  • Máxima cantidad de campos auto-llenados

Contra

  • OCR puede fallar con iluminación/ángulo malo
  • Descarga inicial de ~10 MB (Tesseract)
  • Más lento que escanear un código
C

Combinado: multi-código + OCR

Dos botones en el formulario: "Escanear códigos" (multi-barcode, rápido) y "Leer etiqueta" (OCR, para los campos sin código de barras). El usuario usa el más conveniente según lo que tenga a mano.

Pro

  • Cubre todos los casos
  • El usuario elige según la situación
  • Multi-código muy rápido para campos con barcode

Contra

  • Más complejidad de UI
  • Igual descarga Tesseract (~10MB)
