¿Dónde vive el sistema de requerimientos?

Esta decisión determina el esfuerzo de despliegue y mantenimiento a largo plazo

A

Mismo servidor IT (recomendado)

Una página standalone en servidor:3000/requerimientos.html — igual que MediCalc. Sin infraestructura extra. El formulario es público, el panel admin usa el RBAC existente.

✓ Pros

  • Cero infraestructura extra
  • Ya tienes multer para fotos
  • SQLite compartido
  • Un solo deploy

✗ Contras

  • Depende de que el server IT esté activo
  • Mismo dominio/puerto
B

Servicio separado (otro puerto / subdominio)

App Node.js independiente en reqs.medivalle.co o servidor:3001. Base de datos propia. Totalmente desacoplado del sistema IT.

✓ Pros

  • 100% independiente
  • Puede caerse IT sin afectar reqs
  • Deploy separado

✗ Contras

  • Doble mantenimiento
  • Configuración extra de red
  • Más complejo
C

Google Forms + Sheets + Apps Script

Formulario en Google Forms, respuestas en Sheets, email automático via Apps Script. Cero hosting, gratis.

✓ Pros

  • Sin servidor propio
  • Siempre disponible
  • Gratis

✗ Contras

  • Sin UI personalizada
  • Sin admin integrado
  • Fotos limitadas
  • No se integra con sistema IT
